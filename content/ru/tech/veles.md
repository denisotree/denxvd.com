[Veles](https://github.com/denisotree/veles) — это минималистичный многоагентный фреймворк, который ведёт структурированную память по каждому проекту и становится полезнее с каждой сессией. Он живёт у вас в терминале, работает с любым LLM-провайдером — облачным или локальным — и по ходу работы накапливает знания, навыки и инструменты. Я написал его в первую очередь для себя: я много работаю с LLM, и часто проблема в том, что весь диалог не помещается в контекст, а после сжатия модель забывает важные инсайты — приходится напоминать о правилах, событиях, документах и заново восстанавливать контекст. Я хотел фреймворк, который помогает агентам помнить мои решения, качественно хранит память и при желании работает целиком на локальной модели, не привязанный к одному вендору.

Я много [работаю в терминале]({{< relref "data-analyst-cli" >}}) и люблю, чтобы инструменты были удобными. Veles родился из потребности собрать в одном месте функционал из разных утилит и фреймворков по работе с агентными системами. В этой статье расскажу, зачем я его сделал, что он умеет и как его легко попробовать.

---

### Почему не готовый чат-агент

Действительно, на рынке есть Claude Code, OpenCode, OpenClaw, Hermes (он ближе всего к тому, чего я хотел, но то, как там организован код, вызывает у меня ужас) и десяток облачных агентов — и они правда классные. Но мне хотелось собрать отдельные функции от каждого в одной коробке — идеальный, на мой взгляд, баланс:

1. **Память, а не амнезия.** Чтобы агент помнил контекст проекта между сессиями, а я не объяснял одно и то же заново.
2. **Local-first.** Чтобы память и состояние жили у меня на машине, без телеметрии и облачной синхронизации, — а при желании и сам инференс на локальной модели. Это важно, когда работаешь с чувствительными данными.
3. **Без вендор-лока.** Чтобы один и тот же агент ходил в любую модель — хоть в Claude через API, хоть в локальную через Ollama.
4. **Чистое ядро.** Минимальный код без god-файлов, который не страшно расширять под себя.

Одни готовые инструменты закрывают что-то одно, но не всё сразу. Другие — вроде OpenClaw или Hermes — наоборот, тащат всё из коробки, даже то, что тебе не нужно. Veles я собрал по обратному принципу: минимум в ядре, а всё, что захочется, легко подключается к нему модулями.

### Что умеет Veles

1. **Память, которая накапливается**  
   Это главная фича. После каждой сессии Куратор дистиллирует её в инсайты, правила и дайджесты, которые лежат в `.veles/` прямо в проекте. Дальше агент сам вспоминает релевантные факты и прошлые решения — вы перестаёте гонять по кругу один и тот же контекст. Извлечение инсайтов ловит фразы вроде «всегда делай X» или «никогда не делай Y» и сохраняет их как постоянные правила проекта.

   ![](/tech/veles/tui-hero.gif)

2. **Любой провайдер, без привязки к вендору**  
   OpenRouter, Anthropic, OpenAI, Gemini, Ollama, llamacpp или ваша подписка через `claude` / `gemini` CLI — без API-ключа. Разные типы задач можно отправлять на разные модели: планирование — на модель посильнее, сжатие и извлечение инсайтов — на модель подешевле.

   ```bash
   # Маршрутизация задач по моделям: один раз настроил — и забыл
   veles route set compressor anthropic/claude-haiku-4-5
   veles route show
   ```

3. **Local-first и песочница**  
   Никакой телеметрии и облачной синхронизации — агент видит только каталог активного проекта, а попытки выйти за его пределы (симлинки, `..`) блокируются. На каждый чувствительный вызов (запуск шелла, запись файлов, скачивание по URL) срабатывает «лесенка доверия». А если хочется, чтобы данные вообще не покидали машину, — поднимаете локальную модель через Ollama, и весь цикл, включая инференс, остаётся у вас.

4. **База знаний прямо в проекте**  
   По умолчанию `veles init` разворачивает LLM-Wiki, но можно выбрать и другой шаблон. Дальше агент ведёт проект по выбранному шаблону — или вообще не трогает структуру, если это, например, репозиторий с кодом. В режиме вики ему можно скормить источник, и он превратит его в страницу, на которую потом будет ссылаться в ответах:

   ```bash
   # Добавить источник (PDF, веб-страница, текст) → станет страницей вики
   veles add paper.pdf
   veles run "Что мы знаем о дизайне аутентификации?"
   ```

   ![](/tech/veles/kb-ingest.gif)

5. **Навыки и инструменты, которые накапливаются**  
   Навык — это переиспользуемый блок-промпт (`SKILL.md`), который автоматически становится инструментом агента. Удачный навык можно «повысить» из проекта в глобальный — и он станет доступен везде. Встроенный dedup находит почти одинаковые навыки, пока они не расплодились.

6. **TUI с режимами**  
   `veles tui` открывает интерактивный REPL. Slash-команды показывают всё вживую — `/status`, `/tokens`, `/context`, — а `Shift+Tab` переключает режимы (auto / planning / writing / goal).

   ![](/tech/veles/tui-tour.gif)

7. **Демон и каналы**  
   Veles можно запустить как постоянный демон с HTTP/WS API, к которому подключаются внешние каналы — мессенджеры вроде Telegram или Slack и любые свои интеграции. Из коробки реализован Telegram: вешаете бота и общаетесь с агентом своего проекта прямо с телефона, у каждого пользователя своя постоянная сессия. Остальные каналы легко дописать поверх того же API.

   ![](/tech/veles/tui-daemon.gif)

8. **Длительные задачи**  
   Когда задача не на один заход, есть цели с бюджетом, задания по расписанию и многошаговое исследование.

   ```bash
   # Долгая цель с лимитом по деньгам
   veles goal start "Перенести модуль авторизации на нового провайдера" --max-cost-usd 2.00

   # Глубокое исследование: вопрос разбивается на параллельные подвопросы
   veles research "Какие компромиссы между SQLite и PostgreSQL для этого кейса?"
   ```

### Как попробовать

Veles ставится через [uv](https://docs.astral.sh/uv/) (нужен Python 3.13+):

```bash
# Пакет называется veles-ai, а команда — veles
uv tool install veles-ai
```

```bash
# Ключ — рекомендую OpenRouter: все модели по одному ключу
export OPENROUTER_API_KEY=sk-or-v1-...
```

```bash
# Создать проект и задать первый вопрос
veles init && veles run "Прочитай README и опиши проект"

# Или сразу интерактивный режим
veles tui
```

Если хотите собрать из исходников — `git clone`, затем `uv tool install .` из каталога репозитория.

### Где Veles пока не подходит

Скажу честно: это не замена большим агентным IDE вроде Claude Code или Cursor. Veles — это минималистичное ядро, заточенное под мой рабочий процесс: память, цикл обучения и протокол провайдера, а почти всё остальное (TUI, демон, Telegram, исследования, планировщик) — опциональные модули, которые надо осознанно включать.

Ещё пара честных оговорок. Проект молодой — на момент статьи это версия `0.6.x` в статусе Beta, и он требует Python 3.13+, который стоит ещё не у всех. И, как любой LLM-агент, Veles иногда ошибается: память и «лесенка доверия» сильно помогают, но это инструмент-усилитель, а не автопилот, которому можно слепо доверять. Качество ответов напрямую зависит от выбранной модели.

### Планы и итог

Veles уже стал тем, через что я прогоняю всё больше повседневных задач, — и чем дольше пользуюсь, тем заметнее эффект накопленной памяти. Проект открытый (Apache-2.0) и собран с упором на надёжность: больше 3200 тестов, и все гоняются без сети. В будущих статьях разберу отдельные вещи детальнее: как устроена проектная память и цикл обучения, маршрутизация задач по моделям и как писать собственные навыки и инструменты.
